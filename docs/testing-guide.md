# Testing Guide

## 목적
이 문서는 이 저장소의 검증 흐름을 설명한다.

핵심 정책은 AGENTS.md의 테스트 정책을 따른다.
이 문서는 그 정책을 실제 작업에서 어떻게 해석하고 적용하는지 설명하는 참고 문서다.

## 기본 진입점
기본 검증 진입점은 다음 스크립트다.

./scripts/verify.sh

모든 변경은 우선 이 스크립트를 기준으로 검증한다.

## 검증 방식
`./scripts/verify.sh` 는 저장소의 기본 품질 게이트다.

## 변경 범위에 맞는 검증
변경 범위가 작더라도 관련 검증은 생략하지 않는다.
수정한 파일만이 아니라, 그 변경으로 영향을 받는 경로까지 검증 대상으로 본다.

예:
- 백엔드 비즈니스 로직 수정 시 관련 테스트와 빌드 확인
- 프론트엔드 UI 수정 시 관련 테스트와 린트 확인
- 공통 설정 수정 시 전체 검증 영향 여부 확인

## 실패 처리 원칙
검증 실패 시에는 실패 메시지를 먼저 읽고, 실패 원인을 직접 수정한다.
우회하거나 숨기는 방식으로 통과시키지 않는다.

권장 순서:
1. 실패한 명령과 에러 메시지 확인
2. 실제 원인 식별
3. 원인에 해당하는 최소 범위 수정
4. 동일 검증 재실행
5. 필요한 경우 인접 영향 범위까지 추가 확인

## 피해야 할 대응
다음 방식은 올바른 해결로 보지 않는다.

- 테스트 비활성화
- assertion 완화
- 설정 삭제로 우회
- 임시 mock 또는 hardcode 추가로 통과
- 원인과 무관한 대규모 수정

## 기대하는 결과
작업 종료 시점에는 다음 상태를 기대한다.

- 변경 범위에 맞는 검증이 통과한 상태
- 실패를 숨기지 않은 상태
- 최소 변경 원칙이 유지된 상태
- 재현 가능한 방식으로 검증이 가능한 상태
